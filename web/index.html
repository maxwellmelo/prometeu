What Prometeu does

Six building blocks turn donated capacity into a verifiable, shared inference network.

/v1 inference gateway

Apps, CLIs, bots and UIs call a common chat/completions endpoint with streaming support.

Model pool orchestration

The coordinator sizes GGUF models, selects peers, warms pools and tracks READY / DEGRADED / FAILED state.

Participant nodes

Anyone can contribute controlled CPU / RAM / GPU capacity through a local daemon and dashboard.

Hash-pinned allowlist

Nodes only load curated models with known sha256 hashes. No unverified fallback.

Reciprocity ledger

Serving tokens earns standing. Contributors get more usage headroom while anonymous users keep a baseline floor.

Open observability

Pools, peers, registry state and Prometheus metrics expose what the network is doing.

How it works

A request flows from any client through the gateway down to a verified participant node.

1
Client → Gateway
App / user POSTs to /v1/chat/completions.
2
Gateway
Rate limit + token metering · model / pool routing · reciprocity accounting.
3
Pool / participant node
Verify allowlisted model hash · run sandboxed inference · emit signed receipts.

Start building

Two ways in: call the shared API, or donate capacity by running a node.

Use the API

curl -X POST https://your-coordinator.example/v1/chat/completions \
  -H 'content-type: application/json' \
  -d '{"model":"qwen","messages":[{"role":"user","content":"Hello"}]}'

Run a node

curl -fsSL https://raw.githubusercontent.com/maxwellmelo/prometeu/main/node/install.sh | sudo bash
